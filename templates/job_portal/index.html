Sevendyne Enterprise HRMS

A production-grade Human Resource Management System built with Python & Django over one year by a team of three engineers in our local office. Fork it, run it in one minute, and contribute.

Login to HRMS View on GitHub

HRMS Modules

Clean domain separation — each module maps to a Django app under apps/

Authentication

Role-based login for admins, HRMS clients, and employees. apps/user

Attendance

Check-in, check-out, and attendance registers. apps/employee

Payroll

Salary slips and fixed-point payroll calculation. apps/payroll

Leave Tracker

Leave types, requests, and approval workflows. apps/employee

Also included
  • Recruitment & job portal
  • Candidate management
  • Client & asset tracking
Tech stack

Django 5 · PostgreSQL 15 · Redis · Celery · Docker Compose · pytest · GitHub Actions CI

Repository Architecture

Plug-and-play structure — any company or collaborator can spin up the full stack with docker compose up --build. 

sevendyne_hrms/
├── config/settings/     # base, local, production
├── apps/
│   ├── authentication/  → user
│   ├── attendance/      → employee
│   ├── payroll/
│   └── leave_tracker/   → employee
├── compose/local/       # Dockerfile + entrypoint
├── requirements/        # base, local, production
└── docker-compose.yml   # App + PostgreSQL + Redis

Quick Start

git clone https://github.com/sevendyne/sevendyne_hrms.git
cd sevendyne_hrms
docker compose up --build
# → http://localhost:8000

Migrations and demo users are seeded automatically on first boot. More Sevendyne open-source projects are coming soon — we welcome collaborators.
